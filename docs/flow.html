POC 2 — Oversell prevention strategies

1000 concurrent buyers, 10 units. Race five strategies and see which oversell and which don't.

The bug the naive strategy shows

if (stock >= qty) {         // ← thread A reads 1
    // any work here...     // ← thread B also reads 1, both pass the check
    stock -= qty;           // ← both write, sold=2, stock=-1
}

This bug reproduces once every ~10k requests in staging. Nobody catches it until prod day.

POC 5 — Multi-warehouse routing

Greedy per-line picks the cheapest warehouse for each line and happily splits into 4 shipments. Optimal adds a split penalty and picks what ops actually wants.

Same order, two routers

Score model

score = totalCost + (shipments - 1) × SPLIT_PENALTY

// SPLIT_PENALTY = 100,000 VND ← one place to tune the ops policy

A single ₫82k shipment beats "two ₫37k shipments" because 82k < 37k + 37k + 100k. Ops can change the constant without a code review.

What Kubernetes gives you — and what it doesn't

Gives you

  • Horizontal replicas + HPA
  • Rolling updates with PDB
  • Readiness / liveness health gates
  • ConfigMaps / Secrets
  • Stable Service DNS in front of pods

Does NOT give you

  • Shared state across pods → Redis/DB/Kafka
  • Ordering across pods → idempotency + causal keys
  • Cache warmth on scale-up → prewarm on ready
  • Distributed locks → Redis lock / advisory lock
  • Automatic consistency → reconciliation loops
